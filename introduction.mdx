Aspex is an open-source security toolkit for the MCP ecosystem. It answers two questions every developer using AI coding agents should be asking.

## The problem

You wired an MCP server into Claude Code or Cursor. You copy-pasted the config from a README. The agent now has access to your filesystem, your GitHub, your browser.

**Do you know what that server is actually capable of? Do you know what it did the last time your agent ran?**

## Two tools

<CardGroup cols={2}>
  <Card title="aspex-scan" icon="magnifying-glass" href="/tools/scan">
    Static analysis of every MCP server on your machine. 140+ rules, 0-100 security score, cross-server attack path analysis. Runs fully offline in seconds.
  </Card>
  <Card title="aspex-trace" icon="timeline" href="/tools/trace">
    Runtime audit of your AI agent logs. Reconstructs kill chains, traces instruction provenance, detects credential exfiltration. No proxy, no config change.
  </Card>
</CardGroup>

Together they give you a complete picture: what your servers can do before you run them, and what your agents actually did after.

## Also included

- **`aspex-scan doctor`** - fast pre-flight health check (~2 seconds): env secrets, hardcoded credentials, filesystem grants, network exposure
- **`aspex-attack`** - active red-teaming for security teams who need to probe servers they own

```sh
brew install aspex-security/tap/aspex
```


## Trust

<Note>
  **Aspex never phones home.** All analysis runs fully offline - no data leaves your machine, no account required, no telemetry. Released under Apache-2.0 with a full SPDX SBOM in every release.
</Note>

## Framework coverage

Findings map to OWASP LLM Top 10 2025, MITRE ATLAS, and 20+ CWEs so they slot directly into existing security workflows.
