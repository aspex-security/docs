The aspex VS Code extension brings MCP security scanning directly into your editor. It works in VS Code, Cursor, and any VS Code-based editor (Windsurf, VSCodium, etc.).

## How it works

The extension shells out to the `aspex` CLI and surfaces results as native editor diagnostics.

- **Scan on save** â€” whenever you save an MCP config file (`claude_desktop_config.json`, `.cursor/mcp.json`, etc.), the extension runs `aspex-scan` in the background.
- **Inline squiggles** â€” findings appear as warning or error underlines on the relevant line in the config file.
- **Problems panel** â€” all findings are listed in the standard VS Code Problems panel (`View â†’ Problems`) with severity, rule ID, and a one-line description. Clicking a problem jumps to the offending line.

For findings with an available fix (for example, a hardcoded credential that can be migrated to the macOS Keychain), a lightbulb code action appears inline.

## Install

The extension is not yet published to the VS Code Marketplace. Build it from source:

```bash
# Prerequisites: Node.js 18+, vsce
npm install -g @vscode/vsce

# Clone and build
git clone https://github.com/aspex-security/aspex-vscode
cd aspex-vscode
npm install
vsce package          # produces aspex-vscode-*.vsix

# Install the .vsix
code --install-extension aspex-vscode-*.vsix
```

You also need the `aspex` CLI on your `PATH`:

```bash
brew install aspex-security/tap/aspex
```

Or set a custom binary path in settings (see below).

## Settings

| Setting | Type | Default | Description |
|---|---|---|---|
| `aspex.binaryPath` | `string` | `"aspex"` | Absolute path to the aspex binary. Set this if aspex is not on your `PATH`. |
| `aspex.scanOnSave` | `boolean` | `true` | Run a scan automatically whenever an MCP config file is saved. |
| `aspex.minSeverity` | `string` | `"medium"` | Minimum severity to surface as a diagnostic. One of `low`, `medium`, `high`, `critical`. |

## Command palette

Open the command palette (`Cmd+Shift+P` / `Ctrl+Shift+P`) and search for **aspex**:

| Command | Description |
|---|---|
| `aspex: Scan workspace` | Run a full scan of all MCP configs in the workspace. |
| `aspex: Scan current file` | Scan only the active editor file. |
| `aspex: Show report` | Open the last scan's HTML report in a webview panel. |
| `aspex: Explain finding` | Show the full security advisory (WHY, EXPLOIT, IMPACT, CONFIDENCE) for the finding under the cursor. |
| `aspex: Fix credential` | Migrate a hardcoded token in the current file to the macOS Keychain. |
| `aspex: Clear diagnostics` | Remove all aspex squiggles and Problems panel entries. |

<Note>
All scanning is fully offline. No data is sent anywhere. The extension only invokes the local `aspex` binary â€” there is no telemetry and no account required.
</Note>
