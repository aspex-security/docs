<Steps>

<Step title="Install">

<CodeGroup>

```bash Homebrew
brew install aspex-security/tap/aspex
```

```bash curl
curl -fsSL https://raw.githubusercontent.com/aspex-security/aspex/main/install.sh | sh
```

```bash GitHub Releases
# Download the binary for your platform from:
# https://github.com/aspex-security/aspex/releases
```

</CodeGroup>

</Step>

<Step title="Your first scan">

Run `aspex-scan` to audit every MCP server configured on your machine.

```bash
aspex-scan
```

```
aspex-scan v0.5.3

Clients   Claude Desktop · Claude Code · Cursor
Servers   12 discovered

  filesystem   score 41   HIGH    3 critical findings
  github       score 78   MEDIUM  2 findings
  postgres     score 55   HIGH    4 findings
  ...

Overall score  61 / 100   MEDIUM
Attack paths   2 cross-server kill chains detected

Run aspex-scan --explain for full advisories.
```

Add `--explain` for a full advisory on every finding (WHY, EXPLOIT, IMPACT, CONFIDENCE), or `--html` to generate a shareable report.

</Step>

<Step title="Check what happened">

`aspex-trace` reads native log files from Claude Desktop, Claude Code, Cursor, Windsurf, Cline, and Roo — no proxy, no config change required.

```bash
aspex-trace
```

```
aspex-trace v0.5.3

Sessions   47 analysed
Findings   3 kill chains reconstructed

  CRITICAL  Credential Exfiltration     session a3f2 · 14:02
  HIGH      Prompt Injection Signature  session b91c · 09:45
  MEDIUM    Recon to Credential Theft   session d007 · yesterday

Run aspex-trace session <id> for session forensics.
Run aspex-trace killchain for full MITRE-mapped chains.
```

</Step>

<Step title="Health check">

`aspex-doctor` runs a fast offline health check in ~2 seconds.

```bash
aspex-doctor
```

```
aspex-doctor v0.5.3

Clients      ✓ 3 installed, configs valid
Environment  ✗ GITHUB_TOKEN exposed in shell env (zshrc)
Config secrets  ✗ API_KEY hardcoded in claude_desktop_config.json
Filesystem   ✗ Path grant covers home directory (~/): filesystem server
Network      ✓ All servers use HTTPS

3 issues found. Run aspex-scan fix env to migrate credentials to Keychain.
```

</Step>

<Step title="Interactive launcher">

Run `aspex` with no arguments to open the interactive TUI menu.

```bash
aspex
```

```
  ┌─────────────────────────────────────────────────┐
  │  aspex  MCP Security Suite                      │
  │                                                 │
  │  ▶  scan          Audit all MCP servers         │
  │     trace         Analyse runtime logs          │
  │     attack        Adversarial probe (opt-in)    │
  │     doctor        Quick health check            │
  │                                                 │
  │  ↑↓ navigate · enter select · q quit            │
  └─────────────────────────────────────────────────┘
```

</Step>
